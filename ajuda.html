Sobre o projeto

Este site integra as funcionalidades das Fases 1 e 2 da disciplina Fundamentos de Sistemas Web (PUCRS). Reúne catálogo de Produtos e Serviços, carrossel de Destaques, Cadastro do Cliente com Agendamento de retirada/entrega, validações de formulário, relógio em tempo real e melhorias de acessibilidade.

Como usar o site

  1. No topo há links para Destaques, Cadastro, Produtos, Serviços e Contatos.
  2. O carrossel mostra ofertas com navegação por setas e indicadores. As imagens possuem alt descritivo.
  3. Na seção Cadastro, preencha seus dados. Erros são exibidos abaixo dos campos e anunciados via aria-live.
  4. Escolha o serviço (Retirada ou Tele‑entrega), selecione data e horário (intervalos de 15 min).
  5. Ao clicar em Cadastrar e os dados estiverem válidos, o sistema exibe um delay de 20s indicando validação em andamento. Depois disso, o envio é concluído.

Cadastro e Agendamento

  • Dados pessoais: nome, CPF, telefone, e‑mail, sexo, estado civil.
  • Endereço: CEP, logradouro, número, complemento, bairro, cidade, estado.
  • Serviço: retirada no local ou tele‑entrega. Alerta contextual aparece ao marcar tele‑entrega.
  • Agendamento: escolha de data (type="date") e horário (type="time", step="900").

Validações implementadas

  • CPF com dígitos verificadores (função validarCPF).
  • Telefone no formato (DD) 9XXXX-XXXX e CEP 00000-000 (máscara + regex).
  • Bloqueio de agendamento no passado.
  • Mensagens de erro exibidas somente quando o campo está inválido (Bootstrap .was-validated + .invalid-feedback).

Acessibilidade

  • alt descritivo em todas as imagens (carrossel e produtos).
  • Feedbacks de erro anunciados via aria-live em regiões específicas.
  • Uso de <label>, <fieldset> e <legend> para associação semântica.
  • Navegação por teclado e foco preservados (componentes Bootstrap padrão).

Imagens e carrossel

O carrossel usa classe .carrossel-imagem com altura fixa para uniformizar exibição: 

.carrossel-imagem{ height:380px; object-fit:cover; object-position:center; }
                    @media (max-width:576px){ .carrossel-imagem{ height:240px; } }
  • Tamanho recomendado: 1280 × 380 px (desktop) e 576 × 240 px (mobile opcional).
  • Formato .jpg ou .png, preferencialmente < 200 KB.
  • Coloque os arquivos em assets/ e ajuste os src no index.html caso renomeie.

                        ├─ styles.css      # Estilos do projeto

Observação: mantenha os nomes de arquivos consistentes entre a pasta assets/ e os caminhos usados no HTML.

Publicação no GitHub Pages

  1. Crie um repositório (ex.: minimercado-pucrs).
  2. Envie index.html, styles.css, app.js e a pasta assets/.
  3. Em Settings → Pages, configure:
    Source: Deploy from a branch • Branch: main /root.
  4. Aguarde o endereço do site (ex.: https://seu-usuario.github.io/minimercado-pucrs/).

Perguntas frequentes

As mensagens de erro ficam sempre visíveis. O que fazer?
No styles.css, garanta que .invalid-feedback não esteja forçando display:block. Use:
.invalid-feedback{display:none} e .was-validated .form-control:invalid ~ .invalid-feedback{display:block}.
Quero mudar o tempo do delay antes do envio.
No app.js, em aoEnviarFormulario, altere a variável restante = 20.
Qual o melhor tamanho de imagem para o carrossel?
Exporte em 1280×380 px. Para mobile, opcionalmente 576×240 px.

Solução de problemas

  • Carrossel distorce imagens: padronize as imagens no tamanho recomendado e confira a classe .carrossel-imagem.
  • Formulário não envia: verifique mensagens em vermelho, campos obrigatórios e se a data/horário são futuros.
  • Sem ícones/estilos do Bootstrap: confirme a inclusão do bundle JS/CSS via CDN no index.html.
